## 개요

Tokki는 미국 옥수수/대두 5분각 격자에서 DSSAT 작물 시뮬레이션을 대량
실행하는 Java 오케스트레이터다. 시뮬레이션 입력으로 사용되는 셀별 정보
테이블 (`unit-information_usa-mzsb-47707.csv`, 47,707행 × 14컬럼)을
prep/ 폴더의 8단계 Python 파이프라인으로 만든다.

본 보고서는 이 8단계를 처음부터 재현하는 작업의 진행 상황을 정리한 것.


================================================================================
                           파이프라인 8단계 상태
================================================================================

  Stage | 스크립트                      | 상태       | 비고
  ------+-------------------------------+------------+----------------------
    1   | process_mapspam.py            | 완료       | mapspam2020_USA_wide.csv
        |                               |            | (75,835 행 × 361 컬럼)
  ------+-------------------------------+------------+----------------------
    2   | parse_sol_to_csv.py           | 블록       | US.SOL, cell_id.py 누락
  ------+-------------------------------+------------+----------------------
    3   | extract_resdept.py            | 완료       | rooting_depth_CELL5M.csv
        |                               |            | (75,835 행)
  ------+-------------------------------+------------+----------------------
    4a  | parse_plant_dates.py          | 블록       | GAEZ raster 미확보
        |  (콩 식재일)                   |            | 단, 스크립트는 데이터
        |                               |            | 출처 무관 (ASCII grid면 OK)
  ------+-------------------------------+------------+----------------------
    4b  | nass_maize_planting.py        | 누락       | 스크립트 파일 자체가
        |  (옥수수 식재일)                |            | prep/ 에 없음
  ------+-------------------------------+------------+----------------------
    4c  | smooth_maize_planting.py      | 누락       | 위와 동일
        |  (Gaussian 평활화)             |            |
  ------+-------------------------------+------------+----------------------
   4.5  | (이름 미상)                    | 누락       | wide -> long 변환 +
        |  (포맷 변환)                   |            | ≥25 ha MZ/SB 필터링 +
        |                               |            | MZ/SB 두 작물 join
  ------+-------------------------------+------------+----------------------
    5   | nass_nfert.py                 | 대기       | Stage 4.5 출력이 입력
  ------+-------------------------------+------------+----------------------
    6   | mapspam_irrigation.py         | 대기       | Stage 5 출력이 입력
  ------+-------------------------------+------------+----------------------
    7   | nass_planting_density.py      | 대기       | Stage 6 출력이 입력
  ------+-------------------------------+------------+----------------------
    8   | nfert_recommended.py          | 대기       | Stage 7 출력이 입력


  상태 범례:
    완료 = 스크립트 실행 완료, 산출물 생성됨
    블록 = 스크립트는 있으나 입력 데이터 / 의존성 누락
    누락 = 스크립트 자체가 prep/ 폴더에 없음
    대기 = 선행 단계 출력이 필요해 아직 실행 불가
